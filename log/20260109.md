> [!faq] Docker?
> 컨테이너 가상화 기술을 사용하기 위한 도구
> - =컨테이너 플렛폼
> 	- 컨테이너 엔진
> 	- 컨테이너 런타임
> 클라이언트-서버 모델
> ```mermaid
> graph LR
> Client -- 사용자 명령 전달 --> API&Docker_Deamon -- 결과 전달 --> Client
> API&Docker_Deamon -- 컨테이너 관리 --> HostOS --> API&Docker_Deamon
> ```

이제 컨테이너를 본격적으로 활용해볼 거임
```cmd
docker version # Client, Server의 버전 및 상태 확인
docker info # 플러그인, 시스템 상세 정보 확인
docker --help # 메뉴얼 확인
docker (management command) command # 기본 규격
docker run 이미지명 # management command 가 생략이 가능할 때도 있음
docker container --help
docker rm 컨테이너명/ID # 컨테이너 삭제 명령어
```
처음 서버를 하나 테스트로 띄워보겠음
```cmd
docker run -p 80:80 --name hellonginx nginx
```
다시 없애보겠음 끄는 단축키는 `Ctrl + C` 임
```cmd
docker rm hellonginx
```
