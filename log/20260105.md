> [!faq] 테스트 코드?
> 시스템의 품질과 신뢰성을 검증하기 위한 코드
>  > [!answer] 장점
>  > '코드 변경 -> 기능 점검' 과정 자동화 --> 반복적인 과정을 줄여주어 생산성 향상
>  > 개발자가 시스템에 대한 안정성 확신 --> 유연한 코드 변경 및 리팩터링
>  
> > [!abstract] 심화
> > Unit Test, Integration Test, Regression Test, TDD, BDD 등 다양한 테스트 종류와 방법론 존재

> [!info] PyTest
> 테스트 코드 작성을 위한 Python 라이브러리
> > [!example] vs UnitTest
> > - 간결한 문법
> > 	- assert문
> > 	- 함수 단위 테스트 지원
> > - fixture 지원
> > 	- 테스트 데이터 관리

먼저, `data` dictionary는 이제 필요없으니 삭제
필요한 라이브러리 설치
```cmd
pip install pytest httpx
```
`src` 폴더 안에 `tests`라는 이름의 python package를 만들어야 함
`tests` 폴더 안에는 `__init__.py` 파일이 존재해야 함
`tests` 폴더 안에 파일들의 naming convention을 `test_...` 으로 시작되어야 test코드가 정상적으로 실행 됨
`tests` 폴더 안에 `test_main.py` 파일을 만들고 내용을 작성함
```python
from fastapi.testclient import TestClient
from src.main import app

client = TestClient(app)

def test_health_check():
    response = client.get("/") # 일반적인 반응 호출
    assert response.status_code == 200 # 200번 코드를 출력하는지 확인
    assert response.json() == {"ping": "pong"} # json 결과값 검증
```
그리고 코드를 테스트 해보면
```cmd
pytest
```

다시 원래 하려던거로 돌아오면 `PyTest` 라이브러리로 테스트 코드를 실행하기 위해 명령어를 입력했더니
```cmd
pytest
```
로컬 환경이 아닌 docker 환경에서 실행한 서버라 sql에 접근할 수 없었다.
그리하여 테스트용 DB를 따로 만들 수도 있지만 (실무에서 가장 많이 사용)
나는 강좌에서 쓰는 내용을 따라가기 위해서 도커 안에 `pytest`도 집어넣기로 했다
`docker-compose.yml` 안에 `pytest` 추가해주고
```yaml
version: "3.9"
services:
  mysql:
    image: mysql:8.0
    container_name: mysql
    environment:
      MYSQL_ROOT_PASSWORD: todo
      MYSQL_DATABASE: todos
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-ptodo"]
      interval: 5s
      timeout: 5s
      retries: 10

  api:
    build: .
    container_name: fastapi
    depends_on:
      mysql:
        condition: service_healthy
    ports:
      - "8000:8000"

  test:
    build: .
    depends_on:
      mysql:
        condition: service_healthy
    command: pytest
```
`pytest` 라이브러리를 실행하기 위해 필요한 `httpx` 라이브러리도 `requirements.txt`에 추가한다
