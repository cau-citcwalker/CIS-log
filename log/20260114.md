> [!info] LifeCycle
> = 생애주기
> ```mermaid
> graph LR
> 1["Mount"] --> 2["Update"] --> 3["UnMount"]
> ```
> `Mount`: `component`가 처음 탄생한 순간; 화면에 처음 `rendering`된 순간
> `Update`: `component`가 `rerendering`되는 순간
> `UnMount`: `component`가 화면에서 사라지는 순간

> [!info] useEffect
> `React Component`의 `Side Effect`를 제어하는 새로운 `React Hook`
> > [!faq] Side Effect
> > 부수적인 효과, 파생되는 효과를 의미함
> > ### useEffect
> > ```mermaid
> > graph LR
> > 1["Component 내부의 값 변경"] --> 2["Console에 변경된 값 출력"]
> > 3["Component Mount"] --> 4["Console에 'Mount'라고 출력"]
> > 5["Component Update(Rerender)"] --> 6["Console에 'Update'라고 출력"]
> > 7["Component UnMount"] --> 8["Console에 'Unmount'라고 출력"]
> > ```

`useEffect`를 활용하기 위해서 먼저 `import`를 진행한다
```jsx
import { useEffect } from "react";
```
그리고 예시 코드를 살펴보면
```jsx
useEffect(() => {
    console.log(`Count has been updated to: ${count}`);
  }, [count]);
```
이 코드를 살펴보면 `count`값이 변경될 때마다 `count`값을 출력하도록 되어있다
그리고 뒤에 있는 배열은 변화하는 값을 정하는 배열인데, 이 배열을 의존성 배열 (`Dependency Array; Deps`) 라고 부른다

그런데 여기서 드는 의문점이 하나 있다
그냥 `onClickButton` 이벤트 핸들러 안에 구문들을 넣어도 가능하지 않나?
이 의문점의 답을 강좌에서 바로 알려주었다
`setCount`함수는 비동기로 동작하기 때문에 값이 변경되기 이전에 아래의 구문들이 실행된다
따라서 변경되기 이전의 값들이 실행되는 것을 볼 수 있다

이제 본격적으로 `LifeCycle`을 제어해보려고 한다
먼저 `Mount`이다
```jsx
useEffect(() => {
	console.log("Mount");
}, []);
```
배열 안의 값이 변경되어야 하는데, 안에 값이 없으면 처음 렌더링 될 때만 실행되기 때문이다

다음으로 `Update`이다
```jsx
useEffect(() => {
	console.log("Update");
});
```
이번엔 `deps`를 생략했다
이렇게 하면 처음 `Mount`될 때 실행이 되고, `Rerendering`이 될 때마다 다시 실행이 된다
`deps` 인수를 주지 않아서 모든 변화를 `deps`로 인식하기 때문

만약 `Mount` 시점을 제외하고 `Update`될 때만 실행되게 하고 싶다면
```jsx
const isMount = useRef(false);
useEffect(() => {
	if(!isMount.current) {
		isMount.current = true;
		return;
	}
	console.log("Update");
});
```
이렇게 `useRef`을 활용하여 마운트 시점을 걸러내면 된다

마지막으로 `UnMount`도 제어해보겠다
```jsx
useEffect(() => {
	return () => {
		console.log("UnMount");
	};
}, []);
```
이렇게 하면 `useEffect`가 `Mount`될 때 호출이 된 뒤, `UnMount`될 때 `return` 함수가 호출이 되면서 마무리 된다
