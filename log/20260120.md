> [!faq] Kubernetes?
> `Container`기반 `App`을 자동으로 배포, 확장, 관리해주는 `Container Orchestration`도구
> -> `container`가 많아질 때 알아서 배치하고 살려두고 키워주는 운영 자동화 관리자
> > [!abstract] 핵심 역할
> > ### 1. Deployment
> > 자동으로 배포해주는 기능
> > ### 2. Self-healing
> > `container` 하나가 죽으면 바로 새로 하나 띄워준다
> > ### 3. Auto Scaling
> > 접속자가 폭증할 경우 컨테이너 수를 자동으로 늘리고, 줄어들 경우 자동으로 줄임
> > ### 4. Load Balancing
> > 트래픽을 여러 컨테이너에 고르게 분산시킴
> > ### 5. Non-stop Update
> > 새 버전을 배포할 때 서버를 끊기지 않게 배포할 수 있음
> 
> > [!info] 기본 개념
> > | 개념 | 간단 설명 |
> > | :--: | :-- |
> > | **Cluster** | 서버 묶음 (`Kubernetes`가 관리하는 전체) |
> > | **Node** | 실제 서버 1대 |
> > | **Pod** | `container` 묶음 |
> > | **Container** | Docker container |
> > | **Service** | 외부에서 접근할 수 있도록 해주는 문 |
> > | **Deployment** | `Pod`를 몇 개 띄울지 정하는 설정 |
> > ```mermaid
> > graph TD
> > subgraph Cluster
> > 	subgraph Node
> > 		subgraph Pod
> > 			subgraph Container
> > 			end
> > 		end
> > 	end
> > end
> > ```
